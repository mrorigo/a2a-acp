A2A (Agent-to-Agent Protocol) enables agents to communicate as first-class citizens in modern HTTP-based ecosystems. Zed ACP provides powerful agent capabilities but remains isolated to stdio-based tooling.

A2A-ACP bridges this gap by implementing native A2A v0.3.0 over HTTP — making Zed ACP agents accessible to any A2A-compliant platform or application. This unlocks enterprise integration, agent marketplaces, and seamless workflows across different agent ecosystems.

Universal Agent Access

A2A-ACP makes Claude Code, Codex, and Gemini CLI available to the entire agent ecosystem. No longer confined to local development environments, these agents become accessible via standard HTTP APIs to any A2A-compliant platform, service, or application.

This transforms local Zed ACP tools into enterprise-ready services that can participate in automated workflows, agent orchestration platforms, and multi-agent conversations — all while maintaining their full capabilities and interactive potential.
